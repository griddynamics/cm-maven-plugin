cm:merge

Full name:

com.griddynamics.maven.plugins:cm-maven-plugin:1.3.0-SNAPSHOT:merge

Description:

Merges configuration from different configuration layers according to layers and merge algorithms configuration. Plugin builds hierarchical configuration into several artifacts. Number of artifacts depends on configuration and project structure.

Attributes:

  • Requires a Maven project to be executed.
  • Binds by default to the lifecycle phase: compile.

Optional Parameters

Name Type Since Description
buildDir File - Directory in which plugin will perform all its build operations and to which it will put generated artifacts.
Default value is: ${project.build.directory}.
configDir String - Name of directory under which the configuration layer data is located.
Default value is: config.
dataAlgorithms MergeAlgorithmModel[] - Ordered list of data merge algorithms.
The first applicable algorithm from list will be used to merge data in files.
fileAlgorithms MergeAlgorithmModel[] - Ordered list of file merge algorithms to use.
First one which supports files merge will be used. If non of specified algorithm supports merge, the merge will fail with MergeException.
layers LayerModel[] - Definition of configuration layers.
processors DataProcessorModel[] - Ordered list of data processors available to manage different types of files.
treeAlgorithms MergeAlgorithmModel[] - Ordered list of tree merge algorithms to use.
First one which is able to merge given directories will be used.
type String - Artifacts archive type.
Default value is: zip.
